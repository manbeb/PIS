## Задание №1. Определение процессов

### 🔹 Основной (ключевой) процесс:
**«Организация и проведение учебного процесса в семестре»**

| Параметр | Описание |
|----------|----------|
| **Цель процесса** | Обеспечение своевременного и качественного проведения учебных занятий, контроля знаний и фиксации результатов обучения студентов в соответствии с учебным планом |
| **Инициатор** | Деканат факультета |
| **Входные данные** | • Учебный план на семестр<br>• Списки студентов по группам<br>• Штатное расписание кафедр<br>• Квалификация преподавателей<br>• Учебные поручения деканата |
| **Выходные данные** | • Расписание занятий<br>• Журналы учёта посещаемости и успеваемости<br>• Ведомости экзаменов/зачётов<br>• Отчёт о выполнении учебной нагрузки |
| **Участники** | • Деканат (координация, контроль)<br>• Заведующие кафедрами (распределение нагрузки)<br>• Преподаватели (проведение занятий, аттестация)<br>• Студенты (обучение, сдача контроля)<br>• Учебно-методическое управление (нормативное обеспечение) |
| **Ожидаемый результат** | Все дисциплины семестра проведены в полном объёме, результаты аттестации зафиксированы, нагрузка преподавателей учтена, данные переданы в деканат для отчётности |

### 🔹 Вспомогательный процесс №1:
**«Распределение учебной нагрузки на кафедре»**

| Параметр | Описание |
|----------|----------|
| **Цель процесса** | Рациональное распределение учебных поручений между преподавателями кафедры с учётом их квалификации, категории и доступного фонда времени |
| **Инициатор** | Заведующий кафедрой |
| **Входные данные** | • Учебные поручения от деканата<br>• Список преподавателей кафедры с категориями и ограничениями (кто может читать лекции, вести лабораторные и т.д.)<br>• Предпочтения преподавателей по дисциплинам/группам |
| **Выходные данные** | • График нагрузки преподавателей<br>• Назначения по видам занятий и дисциплинам<br>• Сводная ведомость нагрузки кафедры |
| **Участники** | • Заведующий кафедрой (принятие решений)<br>• Преподаватели (согласование)<br>• Методист кафедры (документальное оформление)<br>• Деканат (утверждение) |
| **Ожидаемый результат** | Нагрузка распределена без нарушений квалификационных требований, все поручения закрыты, преподаватели информированы о своих задачах |

### 🔹 Вспомогательный процесс №2:
**«Сбор и обработка результатов промежуточной аттестации»**

| Параметр | Описание |
|----------|----------|
| **Цель процесса** | Централизованный сбор, верификация и фиксация результатов экзаменов и зачётов для формирования академической отчётности |
| **Инициатор** | Преподаватель, завершивший аттестацию по дисциплине |
| **Входные данные** | • Ведомости с оценками (бумажные/электронные)<br>• Протоколы экзаменационных комиссий<br>• Списки студентов, допущенных к аттестации |
| **Выходные данные** | • Электронные журналы успеваемости<br>• Сводные отчёты по группам/курсам/факультету<br>• Данные для приказа о допуске к сессии/отчислении |
| **Участники** | • Преподаватели (внесение оценок)<br>• Секретари деканата (приём и проверка ведомостей)<br>• Декан (утверждение результатов)<br>• Студенты (ознакомление с результатами) |
| **Ожидаемый результат** | Все оценки внесены в систему, ошибки исключены, сформирована актуальная академическая справка по каждому студенту |

### 📊 Основной процесс: «Организация учебного процесса в семестре»

**Ключевые шлюзы (Gateways):**
- ✅ Проверка: «Соответствует ли квалификация преподавателя виду занятия?» (если нет → перераспределение)

- 🧑‍🎓 Студент

### 📊 Вложенный процесс: «Распределение учебной нагрузки на кафедре»

```
[Старт: Получены учебные поручения]
   │
   ▼
[Анализ поручений: дисциплины, группы, виды занятий, часы]
   │
   ▼
[Фильтрация преподавателей по квалификационным ограничениям]
   │   • Ассистенты/преподаватели → не читают лекции
   │   • Профессора → не ведут лабораторные
   │   • Наличие учёной степени/звания для определённых дисциплин
   │
   ▼
[Формирование предварительного плана нагрузки]
   │
   ▼
[Согласование с преподавателями]
   │
   ▼
{Есть ли возражения/конфликты?}
   ├── Да ──► [Корректировка плана] ──► (возврат к согласованию)
   │
   ▼ Нет
[Утверждение плана заведующим кафедрой]
   │
   ▼
[Передача утверждённого плана в деканат и преподавателям]
   │
   ▼
[Финиш: Нагрузка распределена и зафиксирована]
```

**Элементы BPMN:**
- Start Event: «Поручения получены»
- Tasks: аналитические и согласовательные действия
- Exclusive Gateway: проверка согласования
- Message Flow: обмен данными с деканатом и преподавателями
- End Event: «Нагрузка утверждена»

> 💡 *Рекомендация для визуализации:* Используйте инструменты: Camunda Modeler, bpmn.io, Draw.io или Visio для построения диаграмм в нотации BPMN 2.0.

## Задание №3. Анализ проблем основного процесса (таблица)

| Возможная проблема | Причина | Последствие |
|-------------------|---------|-------------|
| **Узкое место: ручное согласование нагрузки** | Распределение нагрузки происходит в бумажном виде или через разрозненные электронные таблицы, без единой системы учёта доступности преподавателей | Задержки в формировании расписания, конфликты назначений, перегрузка отдельных преподавателей |
| **Дублирование операций: ввод данных об успеваемости** | Преподаватель заполняет бумажную ведомость, затем секретарь деканата вручную переносит данные в электронную систему | Риск ошибок при переносе, увеличение трудоёмкости, расхождения в отчётности |
| **Ручные операции: проверка квалификационных ограничений** | Нет автоматизированной проверки, соответствует ли категория преподавателя виду занятия (лекция/лабораторная) | Назначение некомпетентного преподавателя → нарушение нормативов, риски качества образования |
| **Задержки: сбор ведомостей после сессии** | Преподаватели сдают ведомости в разные сроки, нет автоматических напоминаний и контроля дедлайнов | Задержка в формировании академических справок, проблемы с допуском студентов к следующему семестру |
| **Риск: потеря актуальности данных** | Данные хранятся в разных источниках (журналы кафедр, файлы деканата, личные записи), нет единого реестра | Невозможность оперативно сформировать отчёт по запросу (например, «кто ведёт дисциплину Х в группе Y») |
| **Избыточность ввода: повторное указание одних и тех же данных** | Информация о преподавателе (категория, звание, кафедра) вводится отдельно в расписание, в ведомость, в отчёт по нагрузке | Увеличение вероятности несогласованности данных, рост трудозатрат на поддержку актуальности |
| **Риск: человеческий фактор при распределении** | Заведующий кафедрой распределяет нагрузку «по памяти» или по личным предпочтениям, без аналитики по балансу нагрузки | Неравномерная загрузка преподавателей, снижение мотивации, конфликты в коллективе |
| **Отсутствие трассируемости изменений** | Нет журнала изменений в назначениях и расписании, правки вносятся без фиксации автора и причины | Сложность аудита, невозможность восстановить историю принятия решений при возникновении спорных ситуаций |
