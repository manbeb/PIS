## Задание №2. BPMN-диаграмма основного процесса TO-BE

### 📊 Основной процесс: «Организация учебного процесса в семестре» (TO-BE)
|Дорожка|Роль|Изменения TO-BE|
|---|---|---|
|🎓 Деканат|Планирование, контроль, сбор результатов|Работа в единой ИС, автоматические отчёты, дашборды|
|🏛️ Зав. кафедрой + Методист|Распределение нагрузки|Автоматическая валидация квалификации, электронное согласование|
|👨‍🏫 Преподаватель|Проведение занятий, аттестация|Электронный журнал, электронные ведомости, личный кабинет|
|🧑‍🎓 Студент|Посещение, сдача контроля|Электронная зачётка, онлайн-доступ к оценкам, уведомления|
|⚙️ Информационная система|Автоматизация|Валидация, уведомления, генерация отчётов, хранение данных

**Последовательность действий (TO-BE):**

![[lab4/diagram.svg]]

**Ключевые элементы BPMN TO-BE:**
- ⚙️ **Service Task** — автоматические действия ИС (валидация, рассылка, генерация отчётов)
- 📄 **DataObject** — электронные документы с цифровой подписью
- 🔔 **Timer Event** — автоматические уведомления о дедлайнах
- ✅ **Exclusive Gateway** — автоматическая валидация данных
- 📊 **DataStore** — единое хранилище данных для всех 13 видов запросов

## Задание №3. Сравнительный анализ AS-IS / TO-BE

### 📋 Этапы, которые были исключены:

| Этап AS-IS | Причина исключения в TO-BE |
|------------|---------------------------|
| Ручное формирование поручений в Word/на бумаге | Автоматическая генерация из учебного плана в ИС |
| Рассылка поручений почтой/курьером/лично | Автоматическая рассылка через личный кабинет и email |
| Ручная проверка квалификации преподавателей | Автоматическая валидация бизнес-правилами ИС |
| Бумажный журнал посещаемости | Электронный журнал в реальном времени |
| Бумажная ведомость с подписью | Электронная ведомость с цифровой подписью |
| Ручной приём ведомостей секретарём деканата | Автоматический приём и валидация в ИС |
| Ручное внесение данных в учётную систему | Данные вносятся однократно преподавателем |
| Ручное формирование отчётов в Excel | Автоматическая генерация по 13 видам запросов |

### 📊 Изменение нагрузки на участников:

| Участник | Нагрузка AS-IS | Нагрузка TO-BE | Изменение |
|----------|----------------|----------------|-----------|
| **Деканат** | Высокая (ручной сбор, проверка, ввод, отчёты) | Средняя (контроль, утверждение, анализ дашбордов) | ⬇️ Снижение на 50-60% |
| **Зав. кафедрой** | Высокая (ручное распределение, согласования) | Средняя (утверждение в ИС, контроль валидации) | ⬇️ Снижение на 40-50% |
| **Методист** | Высокая (оформление, проверка, свод таблиц) | Низкая (помощь в ИС, проверка автоотчётов) | ⬇️ Снижение на 70-80% |
| **Преподаватель** | Средняя (бумажные журналы, ведомости, сдача) | Средняя (электронный ввод, но однократный) | ➡️ Без изменений (но проще и быстрее) |
| **Секретарь деканата** | Очень высокая (ручной ввод ведомостей, справки) | Низкая (проверка, печать по запросу) | ⬇️ Снижение на 80-90% |
| **Студент** | Низкая (очное получение справок, доска объявлений) | Низкая (онлайн-доступ 24/7 к оценкам и справкам) | ⬆️ Улучшение доступности без увеличения нагрузки |

### ✅ Устранённые риски:

| Риск AS-IS | Как устранён в TO-BE |
|------------|---------------------|
| Ошибки при назначении преподавателей (квалификация) | Автоматическая валидация бизнес-правилами ИС |
| Ошибки при переносе оценок (бумага → электронно) | Однократный ввод преподавателем, автопередача |
| Потеря бумажных ведомостей и журналов | Электронное хранение с резервным копированием |
| Задержки формирования отчётов (1-3 дня) | Мгновенная генерация по запросам (1-5 минут) |
| Несогласованность данных в разных источниках | Единое хранилище данных для всех модулей |
| Отсутствие трассируемости изменений | Журнал аудита с фиксацией автора, даты, причины |
| Недоступность информации для студентов | Личный кабинет с онлайн-доступом 24/7 |

---

### ⚠️ Новые риски TO-BE:

| Новый риск | Меры минимизации |
|------------|-----------------|
| Зависимость от работоспособности ИС (сбои, отключения) | Резервное копирование, отказоустойчивость, SLA с поставщиком |
| Риски безопасности данных (утечки, несанкционированный доступ) | Ролевая модель доступа, шифрование, 2FA, аудит действий |
| Сопротивление персонала изменениям (нежелание осваивать ИС) | Обучение пользователей, техническая поддержка, поэтапное внедрение |
| Ошибки при первоначальном заполнении реестров (преподаватели, студенты) | Валидация при импорте, двойная проверка, ответственные за данные |
| Цифровое неравенство (не все преподаватели готовы к работе в ИС) | Обучение, инструкции, горячая линия поддержки |
| Юридические вопросы цифровой подписи ведомостей | Согласование с юристами, использование сертифицированных средств ЭП |
