## Задание №1. Анализ стейкхолдеров

### 1. Руководство проектной организации

| Параметр | Описание |
|----------|----------|
| **Интерес к системе** | Получение актуальной аналитической информации о ходе проектов, загрузке персонала и использовании ресурсов |
| **Цели** | Повышение прозрачности бизнес-процессов, своевременное принятие управленческих решений, контроль соблюдения сроков |

## Задание №2. Границы системы

### Что ВХОДИТ в систему:

| Область | Функции |
|---------|---------|
| **1. Управление кадрами (базовое)** | • Учет сотрудников всех категорий (конструкторы, инженеры, техники, лаборанты)<br>• Учет должностей и категорий<br>• Распределение сотрудников по отделам<br>• Учет участия сотрудников в проектах и договорах |
| **2. Управление проектами и договорами** | • Регистрация договоров с заказчиками (номер, заказчик, даты, статус)<br>• Учет проектов с привязкой к договорам (связь многие-ко-многим)<br>• Назначение руководителей проектов и договоров<br>• Формирование состава исполнителей<br>• Отслеживание статусов и сроков выполнения |
| **3. Учет оборудования** | • Инвентарный учет оборудования (номер, тип, модель)<br>• Привязка оборудования к отделам или проектам<br>• Отслеживание статуса (свободно/занято)<br>• История перемещений оборудования между отделами/проектами |
| **4. Учет субподрядчиков** | • Регистрация субподрядных организаций<br>• Учет выполненных работ субподрядчиками<br>• Расчет стоимости работ субподрядчиков |
| **5. Расчетно-аналитическая функция** | • Расчет стоимости договоров на основе выполненных работ<br>• Формирование отчетов по запросам (14 видов отчетов из ТЗ)<br>• Анализ загрузки персонала и оборудования |
| **6. Системные функции** | • Авторизация и аутентификация пользователей<br>• Разграничение прав доступа<br>• Резервное копирование данных<br>• Логирование действий пользователей |

---

### Что НЕ ВХОДИТ в систему:

| Область | Функции (внешние системы) |
|---------|---------------------------|
| **1. Полный бухгалтерский учет** | • Начисление заработной платы<br>• Налоговый учет и бухгалтерские проводки<br>• Формирование официальной бухгалтерской отчетности<br>• *Примечание: система только передаёт данные для бухгалтерии* |
| **2. Кадровое делопроизводство (HR)** | • Приказы о приеме/увольнении<br>• Отпуска, больничные, табель учета рабочего времени<br>• Обучение и развитие персонала<br>• *Примечание: система хранит только базовые данные сотрудников* |
| **3. Электронный документооборот (СЭД)** | • Хранение файлов документов<br>• Подписание документов ЭЦП<br>• Маршрутизация документов на согласование<br>• Архивирование бумажных документов<br>• *Примечание: система хранит только реквизиты договоров, не файлы* |
| **4. Системы проектирования (CAD/CAE)** | • Непосредственное выполнение проектных работ<br>• Создание чертежей и технической документации<br>• Инженерные расчеты |
| **5. Складской учет и снабжение** | • Учет материалов и расходников<br>• Закупки и поставки<br>• Инвентаризация материальных запасов |
| **6. Внешние коммуникации** | • Email-рассылки и мессенджеры<br>• CRM-системы для работы с клиентами<br>• *Примечание: система может отправлять уведомления, но не является коммуникационной платформой* |

---

### Ключевое разграничение: Проекты/Договоры vs Документооборот

| Аспект | В системе | Вне системы (СЭД) |
|--------|-----------|-------------------|
| **Договор** | Реквизиты (номер, дата, заказчик, сроки, статус, руководитель) | Файл договора, подписи, согласования, версии документа |
| **Проект** | Название, сроки, состав команды, статус, привязка к договорам | Проектная документация, чертежи, технические задания |
| **Отчеты** | Формирование выборок и аналитики из БД | Печатные формы с подписями, архивирование |

---

### Внешние системы, взаимодействующие с проектируемой ИС:

| Внешняя система | Входящие данные (в нашу ИС) | Исходящие данные (из нашей ИС) |
|-----------------|----------------------------|-------------------------------|
| **Бухгалтерская система (1С)** | Информация о контрагентах, реквизиты организаций | Расчеты стоимости работ, данные о выполненных работах |
| **HRM-система** | Кадровая информация (ФИО, должность, отдел, дата приема) | Данные о загрузке сотрудников, участии в проектах |
| **СЭД** | Утвержденные договоры (реквизиты), приказы | Отчеты по проектам для архивирования |
| **CAD-системы** | Метаданные о созданных проектных документах | Ссылки на проекты, задания на проектирование |
| **Почтовый сервер** | — | Уведомления о назначении на проекты, напоминания о сроках |

### Процессы, которые можно и нужно автоматизировать:

| Приоритет | Процесс | Обоснование |
|-----------|---------|-------------|
| **Высокий** | Распределение сотрудников по проектам | Исключает конфликты назначений, контролирует загрузку |
| **Высокий** | Учет оборудования | Автоматическое отслеживание статуса и местоположения |
| **Высокий** | Расчет стоимости договоров | Сложные расчеты на основе выполненных работ, минимизация ошибок |
| **Высокий** | Формирование стандартных отчетов | Экономия времени, исключение ручных ошибок |
| **Высокий** | Контроль сроков выполнения проектов | Автоматические уведомления о приближении дедлайнов |
| **Средний** | Планирование загрузки отделов | Оптимизация ресурсов |
| **Средний** | Учет участия субподрядчиков | Контроль затрат |
| **Низкий** | Интеграция с CAD-системами | Требует дополнительных затрат, не критично |
| **Низкий** | Мобильный доступ | Удобно, но не критично для основных пользователей |

## Задание №3. Контекстная диаграмма (DFD уровень 0)

### Описание диаграммы:

**Центральный процесс:**
- **Информационная система проектной организации** (единый процесс)

**Внешние сущности и потоки данных:**

| Внешняя сущность | Входящие потоки (в систему) | Исходящие потоки (из системы) |
|------------------|----------------------------|------------------------------|
| **Руководство организации** | Запросы на аналитику, требования к отчетности | Отчеты о проектах, аналитические сводки, KPI |
| **Начальники отделов** | Заявки на распределение сотрудников, запросы по оборудованию | Отчеты по отделу, информация о загрузке, статусы проектов |
| **Руководители проектов** | Запросы на создание проекта, назначение команды, выделение оборудования | Информация о ходе проекта, отчеты о выполнении, запросы на ресурсы |
| **Сотрудники (исполнители)** | Данные о выполненных работах, запросы на оборудование | Назначения на проекты, информация об оборудовании, личные данные |
| **Бухгалтерия / ПЭО** | Запросы на расчет стоимости, финансовые отчеты | Данные о стоимости работ, финансовая аналитика, расчеты с субподрядчиками |
| **Субподрядные организации** | Информация о выполненных работах, счета | Задания на работы, требования к работам |
| **Администратор системы** | Настройки системы, управление пользователями | Статус системы, логи, уведомления о проблемах |
| **Внешние ИС (1С, HRM, СЭД, CAD)** | Данные из бухгалтерии, HR-системы, реквизиты договоров | Данные для внешней интеграции, отчеты |
